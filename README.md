# Smart Kettle PUI — HTML Demo

Інтерактивне HTML-демо для навчального проєкту з теми **Physical User Interface Design**.

Проєкт демонструє, як розумний чайник може повідомляти користувачу свій стан через світло, ритм, анімацію LED-індикатора та короткі звукові сигнали. HTML-демо відтворює ті самі стани, що й Figma-специфікація та Wokwi FSM-прототип.

## Мета проєкту

Мета — показати чайник не просто як зображення пристрою, а як фізичний інтерфейс зі зрозумілою поведінкою:

- користувач бачить поточний стан пристрою;
- кожен стан має власний візуальний стиль;
- progress, confirm і error не плутаються між собою;
- помилки мають інший ритм, ніж підтвердження;
- READY-confirm спрацьовує тільки при вході в стан готовності.

## Стани

У демо використовується 7 основних станів:

| State            | Опис                                             |
| ---------------- | ------------------------------------------------ |
| `OFF`            | Пристрій вимкнений, індикація неактивна          |
| `IDLE`           | Чайник увімкнений і готовий до запуску           |
| `HEATING`        | Вода нагрівається, LED показує активний progress |
| `READY`          | Вода готова, чайник дає коротке підтвердження    |
| `KEEP_WARM`      | Чайник підтримує температуру у фоновому режимі   |
| `ERROR_NO_WATER` | Недостатньо води для нагріву                     |
| `ERROR_OVERHEAT` | Перегрів, критична помилка                       |

## Мова фідбеку

| Стан             | Світло / звук                                           |
| ---------------- | ------------------------------------------------------- |
| `OFF`            | індикація вимкнена                                      |
| `IDLE`           | м’яке стабільне світло / 1 короткий звук підтвердження  |
| `HEATING`        | плавне breathing-світло / 1 короткий звук підтвердження |
| `READY`          | 2 короткі сигнали + короткий спалах                     |
| `KEEP_WARM`      | дуже повільний спокійний пульс                          |
| `ERROR_NO_WATER` | ритм “довгий + короткий + короткий”                     |
| `ERROR_OVERHEAT` | 3 короткі різкі сигнали з паузою                        |

## Керування демо

У демо є кілька способів перемикання станів:

- кнопки станів зліва;
- кнопки `Prev` і `Next`;
- випадаючий список `Select state`;
- гарячі клавіші `1–7`.

Відповідність клавіш:

```text
1 — OFF
2 — IDLE
3 — HEATING
4 — READY
5 — KEEP_WARM
6 — ERROR_NO_WATER
7 — ERROR_OVERHEAT
```

## Звук

Звукові сигнали реалізовані через браузерний Web Audio API.

Через правила браузерів звук може бути неактивним до першої взаємодії користувача зі сторінкою. Для цього в інтерфейсі є перемикач:

```text
Sound: On / Off
```

Звук використовується тільки як короткий feedback, а не як постійний сигнал.
